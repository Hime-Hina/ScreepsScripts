# Research Guidelines

These rules apply to external docs, local reference clones, and player-code research.

## Pre-Development Checklist

- Search `references/screeps-docs/` before using Screeps API behavior, constants, game loop assumptions, or action ordering.
- Check `docs/references.md` before choosing player repositories to study.
- Record any adopted external idea in a task research note, spec, doc, or ADR.
- Check licenses before copying any code. Prefer learning patterns over copying implementation.

## Guides

| Guide | Applies When |
| --- | --- |
| [Official Screeps Docs](./official-screeps-docs.md) | Using Screeps API, game loop, CPU, memory, actions, or deployment assumptions |
| [Player Code Research](./player-code-research.md) | Studying other Screeps bots or starters |

## Quality Check

External research must leave a local trail:

- Link or local path.
- What was learned.
- What was adopted.
- What was rejected.
- Why the local project constraint differs, if applicable.
